# Методика анализа итогового CSV и пересчета выбросов

Вторая вкладка GUI работает с CSV, который был сохранен на первой вкладке.
Одна строка такого CSV соответствует одному стационарному режиму. При анализе
строки группируются по колонке `Название эксперимента`.

## Суммарное тепло

Для каждого эксперимента суммируется колонка `Количество тепла, кДж`:

```text
Qсум, кДж = Q1 + Q2 + ... + Qn
Qсум, кВт·ч = Qсум, кДж / 3600
```

`Qсум, кВт·ч` выводится как справочная величина для отчета.

## Средние концентрации

Для каждого эксперимента рассчитываются средние значения:

- `O2, %` из колонки `% O2, среднее`;
- `CO, ppm` из колонки `ппм CO, среднее`;
- `NO, ppm` из колонки `ппм NO, среднее`;
- `NO2, ppm` из колонки `ппм NO2, среднее`.

Если рядом есть колонки `N`, среднее считается взвешенно по числу отсчетов:

```text
Xсреднее = sum(Xi · Ni) / sum(Ni)
```

Если колонок `N` нет, используется обычное арифметическое среднее по режимам.

## Пересчет ppm в мг/кВт·ч

Пересчет выполняется по средним концентрациям эксперимента и среднему содержанию
кислорода. Для CO используется коэффициент `1,25`, для NO и NO2 — `2,056`.
Поправочный множитель `10,83 / 11,38` задан методикой.

```text
CO, мг/кВт·ч = 1,25 · COppm · 21 / (21 - O2) · 10,83 / 11,38

NO, мг/кВт·ч = 2,056 · NOppm · 21 / (21 - O2) · 10,83 / 11,38

NO2, мг/кВт·ч = 2,056 · NO2ppm · 21 / (21 - O2) · 10,83 / 11,38
```

Здесь `O2` берется в процентах, `COppm`, `NOppm` и `NO2ppm` — в ppm. Для
корректного пересчета средний `O2` должен быть меньше `21 %`, иначе знаменатель
становится нулевым или отрицательным.

## CSV анализа

При сохранении результата второй вкладки создается CSV с разделителем `;` и
кодировкой UTF-8 с BOM. В нем одна строка соответствует одному эксперименту и
содержит:

- название эксперимента;
- число режимов;
- суммарное количество тепла в `кДж` и `кВт·ч`;
- средние `O2`, `CO`, `NO`, `NO2`;
- пересчитанные `CO`, `NO`, `NO2` в `мг/кВт·ч`.
