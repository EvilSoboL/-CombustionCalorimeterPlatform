# Методика сопоставления XLSX и TXT по времени

Данные из XLSX газоанализатора и TXT-файлов датчиков сопоставляются не по
номерам строк, а по абсолютному времени. Частота записи у приборов может быть
разной, поэтому построчное объединение не используется.

## Временная шкала XLSX

Для XLSX основная временная шкала берется из первой колонки первого листа. По
этим временным меткам строится список доступных минут для выбора стационарных
режимов в GUI.

Границы режима пользователь выбирает в формате `ЧЧ:ММ`. Секунды принимаются
равными `00`. Например:

```text
13:30 -> 2026-06-22 13:30:00
13:35 -> 2026-06-22 13:35:00
```

Перед добавлением режима программа проверяет, что выбранные минуты есть в первой
колонке XLSX.

## Временная шкала TXT

Каждый TXT читается как выгрузка `Oscilloscope Data File`. В заголовке файла
ищется строка `Experiment Time`:

```text
Experiment Time :   22-06-2026 12:50:24
```

Первый числовой столбец секции `Data as Time Sequence` считается временем в
секундах от `Experiment Time`. Для каждого отсчета строится абсолютная временная
метка:

```text
timestamp = Experiment Time + seconds
```

Например, если `Experiment Time = 2026-06-22 12:50:24`, а первый столбец равен
`2400.500000`, то абсолютное время отсчета:

```text
2026-06-22 12:50:24 + 2400.5 с = 2026-06-22 13:30:24.500
```

## Общий диапазон времени

Если загружен только XLSX, режимы проверяются по диапазону XLSX. Если вместе с
XLSX загружены один или несколько TXT, программа вычисляет общий диапазон всех
загруженных источников:

```text
общий старт = max(старт XLSX, старт TXT температуры, старт TXT воды, старт TXT топлива)
общий конец = min(конец XLSX, конец TXT температуры, конец TXT воды, конец TXT топлива)
```

Учитываются только реально загруженные файлы. Режим должен полностью лежать
внутри этого общего диапазона, иначе расчет для выбранного режима будет
отклонен.

## Выбор данных внутри режима

Для каждого режима программа отдельно выбирает данные каждого источника по тем
же абсолютным границам времени:

- XLSX: строки газоанализатора с временными метками внутри режима.
- TXT температуры: отсчеты каналов 1 и 2 внутри режима.
- TXT расхода воды: импульсы расходомера внутри режима.
- TXT расхода топлива: отсчеты аналогового сигнала внутри режима.

Таким образом, файлы не склеиваются построчно. Каждый источник усредняется или
обрабатывается по одному и тому же временному интервалу режима.

## Границы режима

При статистике газоанализатора и средних значениях датчиков используются отсчеты
с временными метками от начала до конца режима включительно.

Для расчета тепла температура на точных границах режима дополнительно
вычисляется линейной интерполяцией по соседним отсчетам TXT температуры. Расход
воды для режима берется как средний расход по числу импульсов за весь интервал.

## Что попадает в CSV

Одна строка итогового CSV соответствует одному режиму. В ней записываются:

- имя эксперимента;
- имя режима;
- начало и конец режима в абсолютном времени;
- длительность режима;
- число отсчетов из каждого загруженного источника;
- рассчитанные средние и производные величины по этому временному интервалу.

Если TXT-файл не загружен, связанные с ним колонки в CSV не создаются.
