# Обработка данных калориметра

GUI-приложение объединяет данные газоанализатора, двух температурных каналов и
расходомера по именованным стационарным временным интервалам (режимам). Для
работы нужен Python 3.10 или новее с Tkinter; внешние пакеты и установленный
Microsoft Excel не требуются.

## Запуск

В Windows дважды щелкните `run.bat` или выполните из папки проекта:

```powershell
python main.py
```

Порядок работы:

1. Выберите XLSX газоанализатора, TXT температуры и TXT расхода.
2. Нажмите **Загрузить и проверить**. Приложение покажет общий диапазон времени.
3. Укажите название эксперимента и коэффициенты калибровки.
4. Добавьте режимы: название, начало и конец. Можно вводить `ЧЧ:ММ:СС` (дата
   берется из XLSX) либо полную метку `ГГГГ-ММ-ДД ЧЧ:ММ:СС`.
5. Нажмите **Рассчитать и сохранить CSV** и выберите итоговый файл.

## Поддерживаемые исходные файлы

Первый лист XLSX должен содержать дату/время в первой колонке, заголовки в
первой строке и измерения ниже. Поддерживаются общие и встроенные строки XLSX.
Колонка `сек` служебная и в статистику не входит. Для всех остальных числовых
колонок рассчитываются среднее, СКО и число отсчетов; поэтому вместе с газовыми
компонентами сохраняются числовые показатели газоанализатора (температуры, КПД,
насос), если они присутствуют.

TXT должны иметь формат выгрузки `Oscilloscope Data File`: строку
`Experiment Time`, затем секцию `Data as Time Sequence`. Поддерживаются UTF-8 и
Windows-1251. Первый столбец числовой секции — секунды от `Experiment Time`.
Файл температуры должен содержать два канала, файл расхода — один:

- канал 1 температуры — температура на входе;
- канал 2 температуры — температура на выходе;
- канал 1 расхода — сигнал расходомера.

Все три файла должны относиться к одной временной шкале. Режим обязан полностью
лежать внутри общего диапазона файлов. Границы режима включаются. Перекрывающиеся
режимы и повторяющиеся названия не допускаются.

## Методика расчета

### Газоанализатор

Для каждого режима выбираются строки XLSX, у которых временная метка находится
между началом и концом режима включительно. По каждой числовой колонке отдельно
пропускаются пустые и нечисловые ячейки и вычисляются:

```text
среднее = (x1 + x2 + ... + xN) / N
СКО = sqrt(sum((xi - среднее)^2) / (N - 1))
```

Используется выборочное стандартное отклонение (`N - 1`). Для одного отсчета
СКО принимается равным нулю. В CSV рядом с каждой парой среднего и СКО пишется
`N`, чтобы было видно фактическое число значений.

### Температура и расход

Напряжение каждого температурного канала переводится в градусы Цельсия одной
линейной калибровкой, задаваемой пользователем:

```text
T, °C = U, В · kT, °C/В + bT, °C
```

`kT` — обязательный коэффициент перевода, `bT` — смещение (по умолчанию 0).
Средние температуры входа и выхода — арифметические средние всех отсчетов в
режиме.

Поскольку предоставленный расходомер также записывает вольты, его калибровка
задается отдельно:

```text
Vdot, л/мин = max(0, (Uflow, В - U0, В) · kV, л/(мин·В))
```

`U0` — сигнал нулевого расхода, `kV` — коэффициент датчика. Отрицательный
результат ограничивается нулем как физически невозможный обратный объемный
расход. Значения `kT`, `bT`, `kV` и `U0` должны браться из паспортов или
экспериментальной калибровки конкретных датчиков; приложение не может надежно
определить их только по файлам напряжений.

### Тепловая мощность и количество тепла

В каждой временной точке температуры расход линейно интерполируется на эту же
метку. Значения на точных границах режима также получаются линейной
интерполяцией. Мгновенная тепловая мощность рассчитывается как:

```text
Vdot_m3_s = Vdot_l_min / 60000
P(t), Вт = ρ, кг/м³ · Vdot_m3_s · cp, Дж/(кг·°C) · (Tвыход(t) - Tвход(t))
```

По умолчанию в GUI заданы свойства воды: `ρ = 1000 кг/м³` и
`cp = 4184 Дж/(кг·°C)`. Их следует заменить, если теплоноситель или условия
эксперимента требуют других значений.

Количество тепла — интеграл мощности по режиму, вычисленный методом трапеций:

```text
Q, Дж = sum((Pi + Pi+1) / 2 · (ti+1 - ti))
Q, кДж = Q, Дж / 1000
Pсредняя, Вт = Q, Дж / длительность режима, с
```

Знак тепла сохраняется: если `Tвыход < Tвход`, результат отрицательный.

## Итоговый CSV

Одна строка соответствует одному режиму. CSV содержит название эксперимента,
название режима, начало, конец, длительность, число исходных отсчетов, средние
температуры, средний расход, среднюю тепловую мощность, количество тепла, а затем
среднее, СКО и `N` каждой числовой колонки XLSX. Разделитель — точка с запятой,
кодировка — UTF-8 с BOM для корректного открытия русских заголовков в Excel.

## Проверка

Автоматические тесты чтения, статистики, формулы тепла и CSV запускаются так:

