# Обработка данных калориметра

GUI-приложение обрабатывает данные газоанализатора по именованным стационарным
временным интервалам (режимам) и, если они загружены, добавляет данные двух
термопар типа K, импульсного расходомера воды и CSV с данными ПЛК.
Обязателен только XLSX газоанализатора. Для работы нужен Python 3.10 или новее
с Tkinter; внешние пакеты и установленный Microsoft Excel не требуются.

## Запуск

В Windows дважды щелкните `run.bat` или выполните из папки проекта:

```powershell
python main.py
```

Порядок работы на вкладке **Обработка данных**:

1. Выберите XLSX газоанализатора. TXT температуры, TXT расхода воды и CSV ПЛК
   можно оставить пустыми.
2. Нажмите **Загрузить и проверить**. Приложение покажет диапазон XLSX или
   общий диапазон времени для всех загруженных файлов.
3. Укажите название эксперимента. Параметры воды, плотности и теплоемкости
   нужны только для соответствующих расчетов; коэффициентов термопары в GUI
   больше нет.
4. Добавьте режимы: название, начало и конец. Начало и конец выбираются в
   формате `ЧЧ:ММ` из выпадающих списков, построенных по первой колонке XLSX.
5. Нажмите **Рассчитать и сохранить CSV** и выберите итоговый файл.

Порядок работы на вкладке **Анализ результата**:

1. Выберите CSV, сохраненный на первой вкладке.
2. Нажмите **Загрузить и проанализировать**.
3. Проверьте сводку по режимам: тепло режима, O2, CO, NO, NO2 и пересчет
   CO, NO, NO2 в `мг/кВт·ч`.
4. При необходимости нажмите **Сохранить анализ CSV**.

## Поддерживаемые исходные файлы

Первый лист XLSX должен содержать дату/время в первой колонке, заголовки в
первой строке и измерения ниже. Поддерживаются общие и встроенные строки XLSX.
Колонка `сек` служебная и в статистику не входит. Для всех остальных числовых
колонок рассчитываются среднее, СКО и число отсчетов; поэтому вместе с газовыми
компонентами сохраняются числовые показатели газоанализатора (температуры, КПД,
насос), если они присутствуют.

TXT-файлы необязательны. Если TXT выбран, он должен иметь формат выгрузки
`Oscilloscope Data File`: строку
`Experiment Time`, затем секцию `Data as Time Sequence`. Поддерживаются UTF-8 и
Windows-1251. Первый столбец числовой секции — секунды от `Experiment Time`.
Файл температуры должен содержать два канала, файл расхода воды — один:

- канал 1 температуры — температура на входе;
- канал 2 температуры — температура на выходе;
- канал 1 расхода воды — импульсный сигнал расходомера калориметра.

CSV ПЛК должен содержать заголовки и временную метку в колонке `created_at` или
в паре колонок `event_date` + `event_time`. Служебные колонки `id`,
`event_date`, `event_time`, `created_at` в статистику не входят. По остальным
числовым колонкам ПЛК рассчитываются среднее, СКО и число значений `N`.

Все загруженные файлы должны относиться к одной временной шкале. Перед добавлением
режима приложение повторно проверяет наличие выбранных минут в первой колонке XLSX.
Секунды для границ принимаются равными `00`. Если загружены TXT, режим обязан
полностью лежать внутри общего диапазона загруженных файлов. Границы режима
включаются. Перекрывающиеся режимы и повторяющиеся названия не допускаются.

Подробная методика сопоставления XLSX, TXT и CSV ПЛК по абсолютному времени:
[METHODOLOGY_TIME_ALIGNMENT.md](METHODOLOGY_TIME_ALIGNMENT.md).

## Пояснение параметров GUI

- `Вода, л/импульс` — паспортный объем воды, который проходит через расходомер
  за один импульс. Если в паспорте указан коэффициент `K` в `имп/л`, то введите
  `1 / K`. Например, `450 имп/л` означает `0,002222 л/импульс`.
- `Свободные концы термопары, °C` — температура клемм измерительного модуля
  или другого холодного спая термопары. По умолчанию стоит `25 °C`. Если модуль
  уже записывает ЭДС, приведенную к свободным концам `0 °C`, поставьте `0`.
- `Плотность воды, кг/м³` и `Теплоемкость воды, Дж/(кг·°C)` используются только
  для расчета тепловой мощности и количества тепла. Если тепло не считается,
  эти поля можно не менять.

## Методика расчета

### Газоанализатор

Для каждого режима выбираются строки XLSX, у которых временная метка находится
между началом и концом режима включительно. По каждой числовой колонке отдельно
пропускаются пустые и нечисловые ячейки и вычисляются:

```text
среднее = (x1 + x2 + ... + xN) / N
СКО = sqrt(sum((xi - среднее)^2) / (N - 1))
```

Используется выборочное стандартное отклонение (`N - 1`). Для одного отсчета
СКО принимается равным нулю. В CSV рядом с каждой парой среднего и СКО пишется
`N`, чтобы было видно фактическое число значений.

### Температура типа K

Напряжение каждого канала переводится в температуру по ГОСТ Р 8.585-2001 для
термопары ТХА(K). Диапазон обратной функции: от `−5,891` до `54,886 мВ`
(`−200…1372 °C`). Линейная пользовательская калибровка не используется.

Термопара измеряет ЭДС между рабочим спаем и свободными концами, а не абсолютную
температуру рабочего спая напрямую. Поэтому программа вводит поправку свободных
концов:

```text
Eраб, мВ = Eизмеренная, мВ + EГОСТ(Tсвободных концов)
Tраб = обратная_НСХ_ГОСТ(Eраб)
```

Если свободные концы около комнатной температуры, небольшое отрицательное
измеренное напряжение не означает отрицательную абсолютную температуру. Оно
может означать, что рабочий спай холоднее клемм измерительного модуля.

Полная методика, коэффициенты и условие компенсации свободных концов:
[METHODOLOGY_THERMOCOUPLE_K.md](METHODOLOGY_THERMOCOUPLE_K.md).

Средние температуры входа и выхода — арифметические средние всех отсчетов в
режиме.

### Расход воды и данные ПЛК

Если файл расхода воды выбран, вода считается по восходящим фронтам импульсного
сигнала с автоматическим гистерезисом. Пользователь задает паспортный объем
одного импульса `Vp`:

```text
Vвода, л = Nимп · Vp
Vdotвода, л/мин = Vвода · 60 / длительность, с
```

Подробности детектора и формулы: [METHODOLOGY_PULSE_FLOW.md](METHODOLOGY_PULSE_FLOW.md).

Если CSV ПЛК выбран, для каждого режима выбираются строки ПЛК по тем же
временным границам. По каждой числовой колонке ПЛК отдельно вычисляются
среднее, СКО и `N`.

### Тепловая мощность и количество тепла

Для режима используется средний расход воды, рассчитанный по числу импульсов.
Температуры на точных границах режима получаются линейной интерполяцией.
Мгновенная тепловая мощность рассчитывается как:

```text
Vdot_m3_s = Vdot_l_min / 60000
P(t), Вт = ρ, кг/м³ · Vdot_m3_s · cp, Дж/(кг·°C) · (Tвыход(t) - Tвход(t))
```

По умолчанию в GUI заданы свойства воды: `ρ = 1000 кг/м³` и
`cp = 4184 Дж/(кг·°C)`. Их следует заменить, если теплоноситель или условия
эксперимента требуют других значений.

Количество тепла — интеграл мощности по режиму, вычисленный методом трапеций:

```text
Q, Дж = sum((Pi + Pi+1) / 2 · (ti+1 - ti))
Q, кДж = Q, Дж / 1000
Pсредняя, Вт = Q, Дж / длительность режима, с
```

Знак тепла сохраняется: если `Tвыход < Tвход`, результат отрицательный.

## Итоговый CSV

Одна строка соответствует одному режиму. CSV всегда содержит название
эксперимента, название режима, начало, конец, длительность, число отсчетов XLSX,
а затем среднее, СКО и `N` каждой числовой колонки XLSX. Если загружены
опциональные TXT и CSV ПЛК, дополнительно появляются средние температуры, число
импульсов и объем воды, средний расход воды, средняя тепловая мощность,
количество тепла и статистика числовых колонок ПЛК. Разделитель — точка с
запятой, кодировка — UTF-8 с BOM для корректного открытия русских заголовков в
Excel.

Разбор повторяющейся информации в предоставленном результате находится в
[CSV_DUPLICATION_ANALYSIS.md](CSV_DUPLICATION_ANALYSIS.md).

## Анализ результата

Вторая вкладка читает итоговый CSV первой вкладки и оставляет каждый режим
отдельной строкой анализа. Для каждого режима берутся его собственные
`Количество тепла, кДж`, `% O2, среднее`, `ппм CO, среднее`, `ппм NO, среднее`
и `ппм NO2, среднее`, а затем CO, NO и NO2 пересчитываются в `мг/кВт·ч`.

Полная методика пересчета: [METHODOLOGY_EMISSIONS.md](METHODOLOGY_EMISSIONS.md).
