# Анализ дублирующейся информации в CSV

Файл результата, предоставленный после первого расчета, содержит 8 строк и 49
колонок. Часть колонок полезна как диагностическая информация, но часть можно
восстановить из других колонок без потери данных.

## Полные дубли

- Колонки вида `<показатель>, N` для всех газовых показателей совпадают с
  колонкой `Отсчетов газоанализатора`. Если все газовые колонки всегда имеют
  одинаковое число валидных значений, эти `N` можно убрать. Если в XLSX бывают
  пустые ячейки по отдельным компонентам, лучше оставить `N`.

## Производные величины

- `Длительность, с` вычисляется как `Конец режима - Начало режима`.
- `Количество тепла, кДж` вычисляется как
  `Средняя тепловая мощность, Вт · Длительность, с / 1000`.
- `ппм NOx, среднее` равно `ппм NO, среднее + ппм NO2, среднее`.

Эти колонки можно убрать из компактного CSV, но в инженерном отчете их часто
оставляют, чтобы файл читался без дополнительных формул.

## Не полные дубли

- `ппм NOx, СКО` нельзя надежно восстановить только из `ппм NO, СКО` и
  `ппм NO2, СКО`, потому что для этого нужна ковариация между NO и NO2.
- Колонки количества отсчетов температуры, расхода воды и ПЛК
  показывают покрытие исходными данными и частоту записи. Это диагностика, а не
  физический результат, поэтому их лучше оставлять хотя бы на этапе отладки.

## Рекомендация

Для рабочего инженерного CSV оставить текущий расширенный состав колонок. Для
финального отчета можно сделать отдельный компактный экспорт без газовых `N`,
`Длительность, с`, `Количество тепла, кДж` и `ппм NOx, среднее`, если эти
значения будут восстанавливаться формулами в отчете.
